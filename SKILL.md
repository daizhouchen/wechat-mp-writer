# 微信公众号内容创作与发布 Skill

本 skill 提供从内容创作到微信公众平台发布的端到端能力。核心理念：**内容为王，流程自动化**。

## 总览：发布工作流

```
创作内容 → 上传图片素材 → 组装图文 → 存草稿 → 预览 → 用户确认 → 发布
```

每一步都有对应的脚本和API调用，skill 会在预览环节暂停等待用户确认，确保不会误发。

---

## 一、首次使用：API 配置引导

如果用户还没有配置微信公众平台API凭据，在执行任何API操作前，先走配置流程。

### 交互式引导步骤

1. **获取凭据**：引导用户登录 [mp.weixin.qq.com](https://mp.weixin.qq.com) → 设置与开发 → 基本配置，获取 AppID 和 AppSecret
2. **设置IP白名单**：用户需要将运行机器的公网IP添加到白名单（可通过 `curl ifconfig.me` 获取）
3. **存储凭据**：将凭据写入项目根目录 `.env` 文件或系统环境变量

```bash
# .env 文件格式
WECHAT_APP_ID=wx1234567890abcdef
WECHAT_APP_SECRET=your_app_secret_here
# 可选：用于预览的接收者微信号或openid
WECHAT_PREVIEW_USER=your_wechat_id
```

### 安全红线

- **绝不**将 AppSecret 硬编码到代码或 SKILL.md 中
- **绝不**将 `.env` 文件提交到 git（确保 `.gitignore` 包含 `.env`）
- access_token 有 2 小时有效期，脚本内置缓存和自动刷新机制

检测方式：运行 `python3 scripts/wechat_api.py check` 验证凭据是否可用。

## 二、内容创作

### 2.1 标题生成

为用户主题生成 **5 个以上候选标题**，涵盖不同风格：

| 风格 | 特点 | 示例 |
|------|------|------|
| 数字式 | 用数字增加具体感 | 「掌握这 5 个技巧，写作效率提升 300%」 |
| 悬念式 | 引发好奇心 | 「为什么 90% 的人都忽略了这个细节？」 |
| 痛点式 | 直击读者痛处 | 「还在加班写报告？是因为你不知道这个方法」 |
| 对比式 | 制造反差 | 「月薪 3000 和月薪 30000 的人，差距在这里」 |
| 提问式 | 引导思考 | 「你的时间管理，真的有效吗？」 |
| 热点式 | 借势热门话题 | 「ChatGPT 爆火后，这个行业最先被颠覆」 |

列出所有候选，让用户选择或修改。标题控制在 22 字以内（微信标题显示限制）。

### 2.2 正文结构

采用 **Hook → Body → CTA** 三段式：

**开头 Hook**（前 3 句话决定读者是否继续）：
- 故事开场：以一个具体场景或人物切入
- 数据开场：用一个反直觉的数据吸引注意
- 提问开场：抛出读者关心的问题
- 痛点开场：描述读者正在经历的困境

**正文 Body**：
- 根据内容类型选择对应模板（见下方）
- 每 3-4 段插入一个图片占位标记 `<!-- IMAGE: 描述 -->`
- 关键观点用**加粗**或引用块强调
- 字数控制在 1500-3000 字

**结尾 CTA**：
- 引导关注：「觉得有用？点个关注，后续更多干货」
- 引导转发：「转发给你觉得需要的朋友」
- 引导留言：「你怎么看？评论区聊聊」
- 引导点赞：「点个"在看"，让更多人看到」

### 2.3 内容类型模板

根据用户需求选择模板，详见 `references/content_templates.md`：

- **干货教程**：问题引入 → 步骤拆解 → 实操演示 → 总结要点
- **观点评论**：现象描述 → 观点亮出 → 论据支撑 → 引发讨论
- **行业分析**：行业背景 → 数据解读 → 趋势判断 → 行动建议
- **产品介绍**：痛点场景 → 解决方案 → 产品亮点 → 使用引导
- **活动推广**：活动亮点 → 参与方式 → 限时优惠 → 报名入口
- **人物故事**：人物引入 → 关键转折 → 成长历程 → 启发总结

### 2.4 摘要（Article Digest）

为每篇文章撰写 120 字以内的摘要，用于分享卡片展示。要求：
- 包含文章核心价值点
- 制造"想点开看"的冲动
- 不要简单重复标题

### 2.5 风格参数化

在创作前确认以下参数（如用户未指定，使用合理默认值）：

- **Tone of voice**：正式 / 轻松 / 专业 / 幽默（默认：轻松专业）
- **目标读者画像**：年龄段、职业、关注点（默认：25-40岁职场人）
- **参考风格**：如用户提供了示例文章，分析其风格并模仿

## 三、排版规范

生成的 HTML 必须遵循以下排版规则（这些规则直接影响手机端阅读体验）：

### 基础样式

```
正文字号：15px（推荐，14-16px 均可）
行距：line-height: 1.8（1.75-2 倍之间）
字色：#3f3f3f（非纯黑，降低视觉疲劳）
段间距：margin-bottom: 20px（段间留白）
两端对齐：text-align: justify
```

### 移动端适配

- 每段 2-4 句话，避免长段落
- 段间空行分隔
- 重要内容用**加粗**、`引用块`、或色块强调
- 图片宽度 100%，避免左右留白

### 配色方案

提供默认配色，用户可自定义：

```
主色调：#1e88e5（蓝色系，适合大多数场景）
强调色：#ff6b35（橙色，用于重点标注）
引用背景：#f7f7f7
分割线色：#eaeaea
```

如用户有品牌色，优先使用品牌色。

### 图片节奏

每 3-4 段插入一张图片，正文中用占位标记：
```html
<!-- IMAGE: 这里描述图片内容，用于后续替换 -->
```

实际发布时，图片必须先上传到微信服务器获取微信域名URL后替换。

## 四、输出格式

### 4.1 微信兼容 HTML

微信公众号编辑器支持的是 HTML 子集，有严格限制。详细兼容性说明见 `references/wechat_html_compat.md`。

关键限制：
- **不支持** `<script>`、`<iframe>`、`<form>`、`<input>` 等标签
- **不支持** `class`、`id` 属性（所有样式必须 inline）
- **不支持** 外部 CSS/JS 引用
- **不支持** `position: fixed/absolute`
- **图片 src** 必须为 `mmbiz.qpic.cn` 域名（微信图片服务器）
- **超链接** 仅支持公众号文章互链和白名单域名，其他外链无效

所有样式通过 `style` 属性内联写入每个元素。

### 4.2 Markdown 输出（可选）

如果用户需要导入秀米、135编辑器等第三方工具，可额外输出标准 Markdown 格式。在 HTML 输出后询问用户是否需要。

## 五、发布工作流

这是核心流程，每一步都有对应的 API 调用。脚本位于 `scripts/wechat_api.py`。

### Step 1：创作内容

按照第二、三、四节的规范，生成：
- HTML 正文
- 标题（用户已确认的）
- 摘要（120字以内）

### Step 2：上传素材

**正文内图片**：
```bash
python3 scripts/wechat_api.py upload_image <image_path>
# POST https://api.weixin.qq.com/cgi-bin/media/uploadimg
# 返回微信域名 URL，替换正文中的图片 src
```

**封面图**：
```bash
python3 scripts/wechat_api.py upload_material <image_path> --type image
# POST https://api.weixin.qq.com/cgi-bin/material/add_material
# 返回 media_id 和 url
```

如果用户没有准备图片，提示用户提供，或建议使用已有素材库中的图片。

### Step 3：添加草稿

```bash
python3 scripts/wechat_api.py add_draft \
  --title "文章标题" \
  --content "<html内容>" \
  --digest "文章摘要" \
  --thumb_media_id "封面图media_id" \
  --author "作者名" \
  --need_open_comment 1 \
  --only_fans_can_comment 0
# POST https://api.weixin.qq.com/cgi-bin/draft/add
# 返回草稿 media_id
```

### Step 4：预览

```bash
python3 scripts/wechat_api.py preview \
  --media_id "草稿media_id" \
  --towxname "用户微信号"
# POST https://api.weixin.qq.com/cgi-bin/message/mass/preview
# 发送预览到用户微信
```

**⚠️ 关键暂停点**：发送预览后，必须停下来告诉用户：

> 预览已发送到你的微信，请在手机上查看实际效果。确认无误后请回复"确认发布"或"可以发"，如需修改请告诉我具体问题。

**绝不**在用户确认前执行下一步。

### Step 5：发布

用户明确确认后（识别关键词：确认发布、可以发、发吧、没问题、OK等）：

```bash
python3 scripts/wechat_api.py publish --media_id "草稿media_id"
# POST https://api.weixin.qq.com/cgi-bin/freepublish/submit
# 返回 publish_id
```

发布后查询状态：
```bash
python3 scripts/wechat_api.py publish_status --publish_id "publish_id"
# GET https://api.weixin.qq.com/cgi-bin/freepublish/get
```

返回文章链接给用户。

## 六、SEO 与传播优化

### 关键词布局

- **标题**：核心关键词放在前半部分
- **首段**：自然融入 1-2 个目标关键词
- **小标题**：使用包含关键词的小标题
- **正文**：关键词密度 2-3%，避免堆砌

### 发布时间建议

| 读者类型 | 推荐发布时间 |
|----------|-------------|
| 职场白领 | 早 7:30-8:30 / 午 12:00-13:00 / 晚 21:00-22:00 |
| 学生群体 | 午 12:00-13:30 / 晚 21:30-23:00 |
| 自由职业 | 上午 10:00-11:00 / 晚 20:00-21:00 |
| 通用 | 晚 20:00-22:00（阅读高峰） |

### 转发引导话术

在文末或评论区引导：
- 「如果你也有同样的经历，转发到朋友圈让更多人看到」
- 「你觉得哪个观点最有共鸣？留言区聊聊」

## 七、合规与风控

发布前自动执行以下检查：

### 敏感词检测

运行检查脚本：
```bash
python3 scripts/compliance_check.py --content "<html内容>"
```

检查项：
- **广告法极限词**：最好、最佳、第一、顶级、绝对、100%、永久 等
- **违禁词**：涉政敏感词、低俗词汇
- **医疗/金融声明**：如涉及健康或理财内容，提醒添加免责声明

如检测到问题，列出具体位置和替换建议，让用户确认修改。

### 原创声明

询问用户是否标记为原创文章（`need_open_comment` 和原创标记）。原创文章享受更高推荐权重，但必须确保内容确实为原创。

### 发布前二次确认

即使用户已在预览阶段确认，在实际调用发布API前再次确认：

> 即将发布文章「{标题}」到你的公众号。这是正式发布，发布后所有关注者可见。确认发布？

## 八、错误处理

### access_token 过期

所有 API 调用失败时检查 errcode：
- `40001`/`42001`：token 过期，自动刷新后重试（最多重试 2 次）
- 刷新逻辑内置于 `scripts/wechat_api.py`

### 素材上传失败

- 检查文件格式（支持 jpg/png/gif，大小限制 10MB）
- 检查网络连接
- 重试 1 次，仍失败则提示用户并给出具体错误信息

### API 限流

- 每日接口调用有次数限制（通常 5000 次/天）
- 遇到限流错误（errcode 45009），提示用户稍后再试
- 建议避免频繁测试，善用预览功能

### 通用错误处理

所有 API 返回的 errcode 非 0 时，脚本会输出：
```
错误码：{errcode}
错误信息：{errmsg}
建议操作：{具体建议}
```

常见错误码对照见 `references/error_codes.md`。
