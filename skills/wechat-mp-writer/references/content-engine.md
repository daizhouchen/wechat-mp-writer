## 1. 三层证据链（核心纪律）

公众号文章不是观点表演秀。一篇能立得住的稿，必须有三层结构：**事实层 → 机制层 → 观点层**，三层缺一不可。

### 事实层

**定义**：可被第三方核证的、带出处的具体陈述。

**判定标准**（任一不满足就不是事实，是话术）：
- 包含至少一个具体名词（人名 / 公司 / 产品 / 年份 / 数字 / 论文标题）
- 能 trace 到 article.json 的 `search_results[]` 某一条
- 在 `facts[]` 里登记，带 `grade` 和 `source_ref`

**反例**：
> DeepSeek 最近很火，很多人都在用，效果据说不错。

**正例**（同主题重写）：
> DeepSeek 在 2025-01-20 发布 R1 模型，72 小时内 App Store 美区免费榜冲到第一，超过 ChatGPT；HuggingFace 上的权重下载量在第一周突破 80 万次（来源：DeepSeek 官方公告 + HuggingFace trending 页面快照）。

**落到 article.json**：每条事实进 `facts[]`，type 取 `data` / `quote` / `event` / `mechanism` 之一，必须填 `source_ref`、`grade`、`fact_anchor`。

---

### 机制层

**定义**：把 ≥2 条事实串起来，解释"为什么"或"怎么做到的"。机制层是公众号文章和"信息汇编"的分水岭。

**判定标准**：
- 段落里显式出现因果链或对比链，不是"因此""所以"一笔带过
- 推导用到的事实在段尾或段中以 `facts_cited` 形式登记
- 推导步骤可被读者反驳（如果不能反驳，那是同义反复）

**反例**：
> R1 性能强，因为它训练得好。

**正例**（同主题重写）：
> R1 在 AIME 2024 上 pass@1 71.0%，超过 o1-mini 的 63.6%（fact-001）。同期论文披露训练只用了 14.8K 条数学样本 + 强化学习（fact-003），样本量是 o1 公开估计的 1/30（fact-004）。把这三件事放在一起看，结论不是"R1 数据多所以强"，而是"小样本 + GRPO 强化学习够用了"——这才是 R1 真正的反常识点。

**落到 article.json**：进 `draft_sections[].html`，且 `facts_cited` 至少列出 2 条。

**关键写法约束**：禁止用"因此""所以""综上"作为机制段落的连接词。改用"把 X、Y、Z 放在一起看""这三件事意味着""反过来推""单看 X 不奇怪，但叠加 Y 就……"等显式推导句式。

---

### 观点层

**定义**：在事实和机制之上，作者本人的判断、立场、预测。

**判定标准**：
- 每条观点 cite ≥3 条事实（在 `facts_cited` 里登记）
- 观点必须可证伪（"R1 会改变开源生态"是观点，"R1 很厉害"不是）
- 观点位置一般在章节末尾或全文末尾，不散落在段落中间

**反例**：
> R1 是中国 AI 的里程碑，让我们一起期待国产大模型的未来。

**正例**：
> 把 R1 的 API 价格（fact-005，是 o1 的 1/27）、开源协议（fact-006，MIT）、HuggingFace 周下载量（fact-007，80 万）三件事叠在一起，我的判断是：未来 6 个月内国内会出现至少 3 家基于 R1 蒸馏的垂直推理模型创业公司。这个判断的边界是——如果监管对开源模型出新规，时间会拖到 12 个月以上。

**落到 article.json**：进 `draft_sections[].html` 章节末段，或全文最后一段。

---

### 三层之间的硬约束

| 约束 | 含义 | 违反后果 |
|------|------|---------|
| 不允许裸观点 | 任何观点段必须 cite ≥3 条事实 | quality_check.py 标 fail |
| 不允许裸事实 | 一段全是事实陈述，没有机制或观点解读 | 这种段最多占全文 30% |
| 推导链必须显式 | 机制段不能用"因此/所以"含混带过 | 改写直到推导可被反驳 |
| 观点必须可证伪 | "X 很厉害"不是观点 | 改成"X 在 6 个月内会 Y"这种可被打脸的判断 |

## 1.5 素材榨干率（v2.1 新增）

**问题描述**：v2 在「$500K vs 月薪 4 万」一文里搜了 14 条 search_results，但 v1 成稿只引用 7 条——一半搜的料没出，读者觉得"内容薄"，写作者也觉得"白搜了"。

**新约束**：

| 指标 | 要求 | 后果 |
|---|---|---|
| 引用率 ≥ 80% | 搜 N 条 search_results，至少 0.8N 条在 facts[] 被引用 | quality_check `material_utilization` 通过 |
| 60% ≤ 引用率 < 80% | 临界值 | quality_check warn |
| 引用率 < 60% | 浪费 | quality_check fail |
| 引用率 < 40% | 严重浪费 | 直接打回重写 |

**两种解法**（任选）：

1. **补 facts 引用未用素材**：把零散没用上的 source 整理进 facts[] + 写进 draft_sections。这通常意味着文章可以加一节（比如"插一刀：Marily Nika 三种 PM persona"就是从原本只一句带过的素材里榨出的整节）
2. **删除未用 search_results**：如果某条素材确实跟最终角度不相关，直接从 search_results[] 里删掉。这样 source_report 不会被它"稀释"

**绝对禁止**：搜了不用 + 不删 + 还在 search_results[] 里挂着——这会污染信任度报告（让读者以为这条料被引用了）。

**与字数的关系**：素材榨干率 ≥ 80% 通常意味着字数会到 medium tier（3500-5500）。如果你只想写 short tier，那就少搜（≤ 6 条 search_results）。

**自检题**：
- 我搜了 N 条素材，进 facts[] 的有几条？
- 没用上的几条，是真不相关，还是我懒得写？
- 删一节比加一节容易，但我是不是错过了一个角度？



主题（用户给的）和角度（你最终要写的切口）之间，是 5 步距离。直接从主题进入写作 = 偷懒，必然写成 wikipedia 摘抄。

### 第 1 步：穷尽

**操作**：用 WebSearch + WebFetch 至少跑 5-8 次不同 query，把主题的官方源、媒体源、行业报告、社交讨论各抓 1-2 条。每条都进 `search_results[]`。

**例子（DeepSeek R1 主题）**：
- query 1: "DeepSeek R1 paper" → arxiv 论文（A 级）
- query 2: "DeepSeek R1 benchmark" → 官方 GitHub README（A 级）
- query 3: "DeepSeek R1 Sam Altman" → 路透社引语（B 级）
- query 4: "DeepSeek R1 训练成本" → 第三方推算（C 级）
- query 5: "DeepSeek 创始人梁文锋" → 财新人物报道（B 级）

### 第 2 步：分维度

**操作**：把搜到的信息按 ≥3 个维度归类。常用维度：技术 / 商业 / 用户 / 历史 / 伦理 / 监管 / 生态。**只从一个维度切等于自废武功**——别人也只能想到那个角度。

**例子**：
- 技术维度：训练方法、benchmark、模型架构
- 商业维度：API 定价、开源协议、对 OpenAI 的价格冲击
- 生态维度：HuggingFace 下载量、二次蒸馏、衍生模型
- 监管维度：美国对国产开源模型的反应

### 第 3 步：找模式

**操作**：在维度交叉处找"反常识"或"还没人说透"的点。

**例子**：技术维度"小样本 RL"+ 商业维度"API 价格 1/27" → 反常识点：通常人会以为"低价靠大规模摊薄成本"，但 R1 是"训练成本本来就低 + 推理优化"。这就是别人少写的角度。

### 第 4 步：生成 ≥3 个角度候选

**操作**：每个角度写一句话 + 三项评分（audience_fit / fact_density / freshness 各 0-5），登记到 `angles[]`。

**例子**：
- ang-1: "R1 的开源策略对国产推理生态的冲击" → fit 4 / density 3 / freshness 5 = 12
- ang-2: "拆开 R1 训练管线看 GRPO 对 PPO 的胜利" → fit 5 / density 5 / freshness 3 = 13
- ang-3: "梁文锋 4 年磨一剑：DeepSeek 创始团队画像" → fit 3 / density 4 / freshness 4 = 11

### 第 5 步：选 + tweaks

**操作**：选总分最高的，但允许加 tweaks（细调切口）。落 `chosen_angle.ref` + `chosen_angle.tweaks`。

**例子**：选 ang-2，tweaks = "聚焦在样本量、训练时长、单卡成本三个具象数字"。

## 3. AI Slop 扩展红线（v1 五铁律之上加 5 条）

v1 SKILL.md §14 已有：无 emoji / 无套话 / 像人写 / 数据真实 / 排版干净。v2 在此之上再加 5 条硬红线。

### 红线 6：框架名禁止出现在正文

**定义**：写作时用的脚手架（事实层 / 机制层 / 观点层 / 波特五力 / 马斯洛 / SWOT 等）必须在脑里、不在纸上。框架是工具不是炫技。

**烂示范**：
> 接下来我们用事实-机制-观点三层结构来分析 DeepSeek R1。首先在事实层……

**好示范**：
> R1 在 AIME 2024 拿到 71%。这个数字单看不奇怪，但如果同时知道它只用了 14.8K 条样本……

### 红线 7：每段 ≥1 个具体名词

**定义**：具体名词 = 人名 / 公司 / 产品 / 地名 / 年份 / 数字 / 论文标题 / 模型名。一段全是"某公司""很多人""一些数据"——重写。

**烂示范**：
> 现在很多大模型公司都在卷推理能力，效果一家比一家好，价格也在不断下探。

**好示范**：
> OpenAI 在 2024-09 发布 o1，价格 60 美元 / 百万 token；DeepSeek 在 2025-01 发布 R1，价格 2.19 美元 / 百万 token；Qwen 在 2025-03 跟进 QwQ，价格在两者之间。半年内推理模型价格跌了 27 倍。

### 红线 8：每篇 ≥1 处反方观点或局限承认

**定义**：单边稿（全唱多 / 全唱衰）一律打回。必须有一段类似"但 X 也指出……"或"这个判断的边界是……"。

**烂示范**：（全文 2000 字都在夸 R1，没有任何 caveat）

**好示范**：
> 但 R1 也不是没短板。Anthropic 研究员 Jack Clark 在 X 上指出，R1 在多模态任务上几乎为零，而 o1 至少能处理图像；R1 长上下文能力也比不上 Claude 3.5 Sonnet。如果业务场景包含图像或长文档，R1 不是首选。

### 红线 9：禁止开场白综合症

**定义**：正文第一句不能是"在数字化时代""随着 AI 的发展""今天我想聊聊""众所周知"等。第一句必须是具体事实 / 场景 / 数字 / 引语。

**烂示范**：
> 在 AI 大模型快速发展的今天，推理能力成为各家厂商竞争的焦点。

**好示范**：
> 1 月 20 日晚上 9 点，DeepSeek 在自己的微信群里甩出一个 GitHub 链接，没有发布会、没有视频、连英文 README 都没翻译。

### 红线 10：禁止总结八股

**定义**：文末不能是"综上所述""总而言之""让我们一起拥抱 AI 的未来"。文末要么是具体下一步、要么是反问、要么是留白、要么是 CTA。

**烂示范**：
> 综上所述，DeepSeek R1 的出现标志着国产大模型迈入新阶段，让我们共同期待 AI 行业的美好未来。

**好示范**：
> 6 个月后回头看这篇文章。如果国内没有出现 3 家基于 R1 蒸馏的推理模型创业公司，欢迎来评论区打我脸。

## 4. Title × Hook × CTA 三件套

### Title（标题）

v1 §4.1 已给 8 种风格。v2 加评分维度 + 候选制：

每个候选打三项分（0-5）：
- **信息密度**：标题里有几个具体名词？
- **吸引力**：会不会让人想点？
- **可信度**：会不会显得标题党？

**生成 ≥3 个候选**进 `meta.title_candidates`，按总分（满分 15）选最高的进 `meta.title`。

**常见标题党陷阱（避雷）**：
- "震惊！X 居然……" — 可信度 0
- "X 的真相，看完你就懂了" — 信息密度 0
- "速看！再不看就晚了！" — 吸引力虚高、可信度极低
- "X 行业内幕大曝光" — 模糊承诺
- "教你三招搞定 X" — 标题密度低、内容必空
- "为什么 X 都在做 Y？" — 通用模板，无具体信息
- "重磅！X 宣布……" — 滥用情绪词
- "我用 X 后，发现了惊人的事" — 个人化伪装的标题党

### Hook（开篇 200 字）

必须做到 3 件事：
1. 抛具体事实 / 数字 / 场景 / 引语（不是抽象判断）
2. 暗示读者读完能学到什么（不是直白承诺）
3. 不出现红线 9 列举的开场白句式

**4 种 Hook 模板**：

- **场景式**：「1 月 20 日晚上 9 点，DeepSeek 在自己的微信群甩出一个 GitHub 链接……」
- **数字式**：「2.19 美元 / 百万 token。这是 DeepSeek R1 的 API 报价，OpenAI o1 的 1/27。」
- **反问式**：「如果训练一个能打平 o1 的推理模型只需要 14.8K 条样本，那 OpenAI 过去两年烧的几十亿美元买了什么？」
- **引语式**：「'a very impressive model'——Sam Altman 在 1 月 26 日的 X 帖子里这样评价 DeepSeek R1。这是他第一次公开承认被中国模型追上。」

### CTA（行动召唤）

不只是"关注我"。三种类型：

- **软 CTA**：「你觉得 6 个月后会有几家 R1 蒸馏创业公司？评论区留你的预测。」
- **中 CTA**：「关于 GRPO 训练方法的详细拆解，看我上一篇《GRPO 是什么》。」
- **硬 CTA**：「关注本号，下周发 R1 vs o1 vs Claude 3.5 Sonnet 的横评。」

**频率约束**：一篇文章 CTA 出现 1-2 次即可，多了让人烦。一般是文末 1 次（硬或中）+ 中段 1 次（软）。

## 6. 写完自检题（强制）

写完一篇文章，在跑 `quality_check.py` 之前先自己过这 10 题。任何一题答 No 必须改。

1. 全文 A 级事实有几条？少于 5 条 → 重新搜集。
2. 有没有任何一段是"裸观点"（无 ≥3 条事实 cite）？
3. 有没有任何一段全是模糊名词（无具体人名 / 公司 / 数字 / 年份）？
4. 有没有 ≥1 处反方观点或局限承认？
5. 正文第一句是不是具体事实 / 场景 / 数字 / 引语？还是开场白综合症？
6. 文末是不是"综上所述/拥抱未来"八股？
7. 框架名（事实层/机制层/波特五力等）有没有漏在正文里？
8. 机制段是不是用了"因此/所以"含混带过，而不是显式推导？
9. 标题候选生成了 ≥3 个吗？是按 9 分制选的吗？
10. CTA 是不是出现 ≤2 次？类型是不是和文章定位匹配？

10 题全过 → 进 `quality_check.py` 量化扫描。
